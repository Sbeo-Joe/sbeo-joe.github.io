앤트로픽이 <mark>2026년 3월</mark>, Claude가 사용자의 컴퓨터를 직접 제어해 작업을 완료하는 AI 에이전트 기능을 정식 출시했다. '리서치 프리뷰' 딱지가 붙었지만, AI가 인간의 컴퓨터를 조작하는 시대가 실험 단계를 넘어 실제 서비스로 진입했다는 신호다.

## 핵심 포인트

- Claude Code가 웹 브라우저·파일 시스템·터미널 등 컴퓨터 전반을 직접 제어하는 기능을 탑재했다
- 앤트로픽은 "보안 장치가 절대적이지 않다"며 사용 시 주의를 공개적으로 경고했다
- Microsoft·Google·Salesforce 등 경쟁사의 AI 에이전트 출시가 연이어 예고된 상황에서 선제적 포지셔닝이다

## AI 에이전트: 채팅에서 행동으로

지금까지 AI는 주로 텍스트를 생성하고 질문에 답하는 역할이었다. Claude가 "이 보고서를 요약해줘"라는 요청에 텍스트를 출력하는 것이 전형적인 사용법이었다. 이번 업데이트는 그 경계를 완전히 바꾼다.

이제 Claude는 브라우저를 열고, 웹 페이지를 탐색하고, 파일을 생성하고, 코드를 실행하고, 터미널 명령을 입력하는 등 <span class="hl-blue">인간이 컴퓨터로 할 수 있는 거의 모든 작업</span> 을 자율적으로 수행할 수 있다. "경쟁사 가격 조사해서 스프레드시트로 정리해줘"라고 하면, Claude가 브라우저를 열고 검색하고 데이터를 취합해 파일로 저장한다. 사람은 지시만 하면 된다.

이를 업계에서는 'AI 에이전트(agentic AI)'라고 부른다. LLM(대규모 언어 모델)이 단순 대화 상대에서 <span class="hl-blue">자율적 실행자</span> 로 격상되는 패러다임 전환이다.

## 기술 구현과 보안 딜레마

컴퓨터 제어 AI 에이전트 구현의 핵심 과제는 '화면 이해'와 '행동 결정'의 조합이다. Claude는 화면 스크린샷을 시각적으로 분석해 현재 상태를 파악하고, 다음 클릭·입력·스크롤 동작을 결정한다. 이 과정은 사람이 컴퓨터를 쓰는 방식과 구조적으로 동일하다.

그러나 앤트로픽이 직접 "보안 장치가 절대적이지 않다"고 경고한 데는 이유가 있다. AI 에이전트가 컴퓨터를 자율적으로 제어하면, <span class="hl-red">악의적 웹 콘텐츠나 프롬프트 인젝션 공격</span> 에 노출될 수 있다. 예를 들어, 웹 페이지에 숨겨진 텍스트가 "이 파일을 삭제하라"는 지시를 담고 있을 경우, AI가 이를 정상 명령으로 오인해 실행할 위험이 있다. 이를 '간접 프롬프트 인젝션(indirect prompt injection)'이라 부른다.

앤트로픽은 이를 방지하기 위해 의심스러운 행동 시 사용자 확인을 요청하는 중단점을 설계했지만, 완벽한 차단은 불가능하다고 인정한다.

## 경쟁 구도: AI 에이전트 전쟁의 서막

| 기업 | AI 에이전트 제품 | 현재 상태 |
|------|----------------|-----------|
| Anthropic | Claude Code (컴퓨터 제어) | 리서치 프리뷰 출시 |
| OpenAI | Operator | 출시 중 |
| Google | Project Mariner | 리서치 단계 |
| Microsoft | Copilot Agent | Windows 통합 |
| Salesforce | Agentforce | 기업 고객 출시 |

<span class="hl-blue">앤트로픽이 선제적으로 리서치 프리뷰를 출시한 것</span> 은 기술 선도 이미지 확보와 실사용 데이터 수집이라는 두 마리 토끼를 잡기 위한 전략이다. 실사용 데이터는 에이전트 성능 개선에 가장 강력한 연료이기 때문이다.

> "AI가 '조언자'를 넘어 '실행자'가 되는 세계는, 생산성과 위험이 동시에 급상승하는 세계이기도 하다."

장기적으로 AI 에이전트가 보편화되면, 지금의 RPA(로봇 프로세스 자동화) 시장 — 글로벌 규모 약 <mark>130억 달러</mark> — 이 AI 에이전트로 상당 부분 대체될 가능성이 높다. UiPath·Automation Anywhere 등 RPA 전문 기업들에게는 구조적 위협이다. 반면 AI 에이전트를 활용하는 기업은 단순 반복 업무 인력 비용을 대폭 절감할 수 있다.

## 앞으로 주목할 변수

AI 에이전트의 실질적 확산 속도는 결국 두 가지에 달려 있다.

첫째, 보안 문제를 얼마나 빠르게 해결하느냐다. 기업 환경에서 AI 에이전트가 사내 시스템을 자율 조작하려면 IT 보안 팀의 신뢰를 얻어야 한다. 현재 리서치 프리뷰 단계에서 보안 취약점이 어느 정도 드러나고 개선되느냐가 기업 도입 시점을 결정한다.

둘째, <span class="hl-blue">멀티 에이전트 협업</span> 이 가능해지는 시점이다. 여러 AI 에이전트가 역할을 나눠 복잡한 업무를 병렬로 처리하는 구조가 구현되면, 생산성 혁신의 폭이 달라진다. 앤트로픽은 이미 멀티 에이전트 오케스트레이션 기능을 개발 중임을 시사한 바 있다.

Claude Code의 컴퓨터 제어 기능은 시작에 불과하다. AI가 단순히 조언하는 것을 넘어 실제로 컴퓨터를 움직여 일을 처리하는 세계 — 그 첫 장면이 <mark>2026년 3월</mark>에 열렸다.
