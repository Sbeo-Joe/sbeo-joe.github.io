GitHub Copilot이 사용량 기반 과금으로 전환한다. 같은 날 Warp 터미널이 오픈소스로 전환했고, Ghostty는 GitHub를 떠났다. 개발자 도구 시장에서 하루 만에 세 가지 중요한 변화가 일어났다.

## Key Takeaways

- GitHub Copilot, 월정액 구독에서 <span class="hl-blue">실제 사용량 기반 과금</span>으로 전환
- Warp 터미널, 전체 코드베이스를 오픈소스로 공개
- Ghostty·BookStack 등 오픈소스 프로젝트들이 GitHub 이탈 중 — AI 학습 데이터 활용 우려

## GitHub Copilot의 전략 전환

Ars Technica에 따르면, GitHub는 Copilot 사용자들에게 실제 AI 사용량에 기반한 요금을 부과하는 방식으로 전환한다. 기존의 월 정액 구독 모델에서 벗어나 '쓴 만큼 낸다'는 방식이다.

이 변화는 AI 코드 어시스턴트 시장 전반에 시사점을 준다. 정액제는 헤비 유저에게 유리하지만, 가끔만 쓰는 개발자들에게는 과다 지출 우려가 있었다. 사용량 기반 모델은 <span class="hl-blue">라이트 유저의 진입 장벽을 낮추는</span> 동시에, 실제 사용량이 높은 팀의 비용을 높일 수 있다.

경쟁사인 Cursor, Windsurf, JetBrains AI 등과의 경쟁이 치열해지는 상황에서, GitHub는 가격 유연성으로 시장 점유율을 지키려는 것으로 보인다.

## Warp 오픈소스 전환 — AI 터미널의 새 국면

같은 날, AI 기반 터미널 앱인 Warp가 전체 코드베이스를 오픈소스로 공개했다. Warp는 기존 터미널에 AI 자동완성, 명령어 설명, 워크플로우 자동화 기능을 더한 개발자 도구다.

오픈소스 전환의 배경에는 커뮤니티 주도 성장 전략이 있다. 개발자 도구 시장에서 오픈소스 공개는 신뢰 구축과 빠른 확산의 검증된 방법이다. VS Code, iTerm2 등이 오픈소스 기반으로 개발자 시장을 장악한 경로와 같다.

> "오픈소스화는 제품을 포기하는 것이 아니라, 개발자 커뮤니티를 공동 개발자로 만드는 것이다."

## GitHub 이탈 트렌드 — 플랫폼 신뢰 문제

같은 시기, 오픈소스 프로젝트들의 GitHub 이탈 움직임도 주목받고 있다.

- **Ghostty**: 인기 터미널 에뮬레이터. GitHub를 떠나 자체 호스팅으로 이전
- **BookStack**: 오픈소스 위키 플랫폼. GitHub 이슈에서 Codeberg 이전을 공식 발표

이들의 공통된 이유는 Microsoft 소유의 GitHub에 대한 <span class="hl-red">플랫폼 의존 위험</span>과 AI 학습 데이터 활용에 대한 우려다. GitHub Copilot이 오픈소스 코드를 학습 데이터로 활용했다는 논란이 오래전부터 있어왔다. 코드를 GitHub에 올리는 것이 곧 AI 학습에 기여하는 것이라는 인식이 퍼지면서, 일부 프로젝트들이 대안 플랫폼을 선택하고 있다.

| 프로젝트 | 이전 전 플랫폼 | 이전 후 플랫폼 |
|---------|------------|------------|
| Ghostty | GitHub | 자체 호스팅 |
| BookStack | GitHub | Codeberg |

## Lovable, 모바일 바이브 코딩 시대 선언

같은 날 Lovable은 iOS/Android 앱을 출시했다. Lovable은 자연어 프롬프트로 웹앱을 생성하는 '바이브 코딩(vibe coding)' 플랫폼으로, 이번 모바일 앱 출시로 이동 중에도 웹앱을 만들 수 있게 됐다.

<span class="hl-blue">바이브 코딩</span>은 코드를 직접 작성하지 않고 AI에게 의도를 설명해 앱을 만드는 방식이다. 전통적인 개발자가 아닌 사용자들도 웹앱을 만들 수 있는 시대가 열리고 있으며, 이는 개발자 도구 시장의 타깃 고객층을 근본적으로 확장하는 변화다.

## 개발자 도구 시장의 새 질서

하루 안에 일어난 이 변화들은 하나의 흐름을 가리킨다. AI 코딩 어시스턴트 시장이 성숙기에 접어들면서 가격 전략, 오픈소스 정책, 플랫폼 신뢰 문제가 경쟁의 핵심 변수가 되고 있다.

개발자들이 도구를 선택할 때 고려하는 요소가 기능만이 아니라 <span class="hl-blue">플랫폼 철학</span>과 <span class="hl-red">벤더 종속 위험</span>으로 확장되고 있다는 것이 이번 변화들의 공통된 메시지다. GitHub가 Copilot 수익화를 강화할수록, 개발자 커뮤니티의 플랫폼 다변화 움직임도 함께 가속될 것이다.
